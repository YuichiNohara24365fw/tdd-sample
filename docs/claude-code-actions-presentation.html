💡 実践的な使用例

📝 基本的なテスト実行

GitHub Issue作成例:
タイトル: [CLAUDE] カバレッジテスト実行 本文: 全テストとカバレッジレポートを実行してください → 自動実行: npm test, npm run test:coverage (GitHub Actions) → 結果: 定型レポートをIssueにコメント

🔄 TDD新機能実装

完全TDDサイクル実行:
タイトル: [CLAUDE] TDD新機能実装 本文: ArrayUtilsクラスを以下の仕様でTDD実装してください： - unique(array): 重複要素除去 - sum(array): 配列要素合計 - average(array): 平均値計算 TDDサイクル（Red→Green→Refactor）で実装してください。 → AI支援実行: 1. Claude Codeが失敗するテストコードを提案 (Red) 2. Claude Codeが最小実装コードを提案 (Green) 3. Claude Codeがリファクタリング案を提案 (Refactor) ※ 開発者が各段階で提案を確認・実装

🔒 セキュリティレビュー

包括的レビュー実行:
タイトル: [REVIEW] セキュリティチェック 本文: プロジェクト全体をセキュリティ観点でレビューしてください： - 脆弱性スキャン - エラーハンドリング確認 - 入力値検証のチェック → 自動実行: ESLint、定型品質チェック (GitHub Actions) → AI支援: Claude Codeがセキュリティ観点での改善提案を生成 → 結果: 定型レポート + AI による改善提案

🤖 AIと自動化の役割分担

🤖 AI（Claude Code）の役割

  • 要求分析: Issue内容の理解・解釈
  • 設計提案: アーキテクチャ・パターンの提案
  • コード生成: テストコード・実装コードの作成
  • 改善提案: リファクタリング・最適化案
  • レビュー: セキュリティ・品質観点での分析
  • ドキュメント: コメント・説明の生成
実行方法: 開発者がClaude Codeと対話して実行

⚙️ 自動化（GitHub Actions）の役割

  • テスト実行: npm test, jest の実行
  • カバレッジ分析: テストカバレッジ測定
  • リント実行: ESLint による品質チェック
  • ビルド実行: アプリケーションビルド
  • レポート生成: 定型フォーマットでの結果出力
  • 通知送信: GitHub API経由でのコメント投稿
実行方法: Issue作成で自動トリガー
🎯 協力関係: Claude Code（AI）が戦略・設計・コード提案を行い、GitHub Actions（自動化）が定型的な実行・検証・レポーティングを担当。 開発者は両者の結果を統合して最終的な実装を決定します。
